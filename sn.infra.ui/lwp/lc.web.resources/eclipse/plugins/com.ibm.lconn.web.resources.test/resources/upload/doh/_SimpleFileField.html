Simple File Field Tests

Once a test has finished, the next test will begin. After all tests have finished, click the button at the bottom of the page to complete the test and go to the log page.

Note: Ignore the timing from D.O.H. page for this test. It is regarded as asynchronous and will only report as a few milliseconds.

Test Name Description Expected Result Test Widget
testAddFile After 1 second, adds a file named "myfilename.txt" to the list You will see a browse input for 1 second, then myfilename.txt with a text file icon and an x
testRemoveFile Removes a file named "myfilename.txt" from the list A file named "myfilename.txt" will be immediately added to the list (you may see a small flash as it is added). After 1 second, the file will be removed and you should see the file input again.
testRename Renames a file A file named "myfilename.txt" will be renamed to "myNEWfilename.txt" after 500 ms
testRemoteDuplicateNoActions Creates a file that lives on the server A file named "remoteDuplicate.txt" will be added to the list. Any files with "remote" in the name will be marked as a remote duplicate and an error message will be shown. There should be no remediation actions available.
testRemoteDuplicateAllowReplace Creates a file that lives on the server A file named "remoteDuplicate.txt" will be added to the list. The "replace" action should be available and will be executed after 1 second.
testRemoteDuplicateAllowRename Creates a file that lives on the server A file named "remoteDuplicate.txt" will be added to the list. The "rename" action should be available and will be executed after 1 second. An edit box will appear and after 500 ms, the name will be changed to something valid and the error message should disappear.
testRemoteDuplicateAllowBoth Creates a file that lives on the server A file named "remoteDuplicate.txt" will be added to the list. The "rename" and "replace" actions should be available.
testInvalidCharacters Creates a file with invalid characters in its name A file named "i:nvalid.txt" will be added to the list. It will then be renamed to "inv>alid.txt", and then the invalid characters will be replaced. After 1 second, the original name will be reverted and the error message should reappear.
testLongFilename Creates a file a name which is too long for the serveer to support Will create a file with a name that is 64 characters long for a server that only supports 40 characters or less in names. After 1 second, the name will be truncated.
testLongInvalidDuplicate Creates a file a name which is too long for the serveer to support with invalid characters that is also a remote duplicate. Will create a file with the name "remoteInvalidFile&.doc". The maxlength will be set to 10, so you should only see invalid character and max length violation errors.
testLongDBCS Create a file with a DBCS file name that is too long Creates a file with less than 10 characters that takes more than 10 bytes to trigger a max length violation.
testAllRTL Tests a filename that is RTL  
testMixedDir Tests a filename with LTR + RTL characters
